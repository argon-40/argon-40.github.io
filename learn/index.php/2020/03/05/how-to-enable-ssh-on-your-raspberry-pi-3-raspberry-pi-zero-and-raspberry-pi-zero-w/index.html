How to enable SSH on your Raspberry Pi 3, Raspberry Pi Zero, and Raspberry Pi Zero W

Byadmin

Mar 5, 2020

Do you want to perform software updates to your latest Raspberry Pi 3 project without dismantling its current setup? Or do you just want to access your Raspberry Pi but don’t have a dedicated monitor and keyboard for your project? This can all be done via the SSH protocol, a built-in feature in all Raspberry Pis with a Raspbian OS.

As you already know, the Raspberry Pi 3, Raspberry Pi Zero and Raspberry Pi Zero W are nifty single-board computers capable of many DIY projects and applications. Most of these projects run “headless” (which means that there is no graphical display, monitor, and peripheral devices connected to your Pi). Commonly, you access and administer your Raspberry Pi via VNC (Virtual Network Computing) or SSH—both of which are available and pre-installed in your Raspbian.

The SSH protocol (also referred to as Secure Shell) is a method for secure, remote login from one computer to another. It provides several alternative options for strong authentication and protects communications security and integrity with strong encryption. It is a secure alternative to the unprotected login protocols, such as rlogin and telnet, and insecure file transfer methods such as FTP.

This tutorial will guide you in accessing your Raspberry Pi 3 via SSH.

Step 1 – Know your Raspberry Pi’s IP address

If you have access to the graphical user interface via a monitor connected to your Raspberry Pi, you can find this by typing the command below

$ ifconfig

into the terminal software.

Special OPTION: Linux Only

In case you’re already running headless at this point, you can find the IP addresses of the devices connected to the same network as you by installing nmap (Linux only).

After installing nmap, type in the terminal:

$ sudo nmap -sP 192.168.1.0/24

wherein 192.168.1.0 is your router’s IP address. This may vary depending on how many networks you have. If somehow the command fails, try changing 192.168.1.0 to 192.168.2.0 and so on.

Step 2 – Enable SSH on your Raspberry Pi

Option 1

The SSH option is “disabled” by default in your Raspberry Pi. To enable SSH, you just have to create a file named ssh in the boot partition of your SD card. The content of the file does not matter. Take note that it has no suffix.

Option 2

If you have access to a graphical user interface, you can enable it using the terminal. Type,

$ sudo raspi-config
  1. Select Interfacing Options


2. Select SSH


3. Choose ”Yes”

4. Select ”Ok”


Step 3 – Setup your Client (Mac, Linux, and Windows)

Accessing your Raspberry Pi on your PC via SSH depends on your PC’s operating system. Linux and Mac OS have SSH natively built into their system, but Windows requires you to install third-party SSH clients.

You will be prompted for a password for your Raspberry pi: the default password is “raspberry” and default user is “pi” for the first time it’s launched. We recommend that you change the default password.

Linux and Mac

For Linux and Mac, just type,

$ ssh pi@192.168.2.109

wherein “192.168.2.109” is the IP address of your Raspberry Pi. Just change the IP address to your own and you’ll be ready to login to your pi using SSH!

Windows

You will need to install a third-party SSH client in order to use SSH on Windows. For this tutorial, we are going to use puTTY. You can download putty.exe for basic SSH here.

  1. Double-click on the putty.exe program.
  2. Enter the necessary settings

* ip address
* port
* connection type
3. Click Open to start session.
4. Accept the connection
5. Input username and password
